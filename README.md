# Naam Jaap Counter v2.3 — Auth Safe

A mobile-first Progressive Web App for digital mantra/Naam Jaap counting, Google Sheets synchronization, verified Google user login, and pseudonymous device/activity history.

## Security model

- Google Identity Services button
- Google ID token verified by the Apps Script backend
- Token sent by POST body, not URL query string
- Token kept in browser `sessionStorage`, not `localStorage`
- Raw device ID never stored in Google Sheets
- Device Key generated by server-side SHA-256 hashing with a private pepper
- User-controlled Sheet text protected against formula injection
- Old unauthenticated GET action router must be disabled

## Important configuration

The same Google Web OAuth Client ID must be configured in:

1. Frontend `config.js`
2. Apps Script Script Properties through `setupSecureAuthentication()`

The package intentionally contains a placeholder Client ID because the real ID must be created in the owner's Google Cloud project.

## Existing APIs used

The secure Apps Script router calls the existing project functions:

- `getDashboard()`
- `addCount(number)`
- `saveMantra(mantra)`
- `resetToday()`
- `resetAll()`

The History screen reads the authenticated user's new audit history from `User_Activity`.

## Verification scope

This implementation is designed for a small, low-volume Apps Script application. It uses Google's `tokeninfo` endpoint because Apps Script cannot directly use the usual server client libraries. For a high-traffic public production service, use Cloud Run or Cloud Functions with an official Google Auth library.

## v2.4 Fast Tap Sync

- Counter updates instantly on tap (optimistic UI).
- Rapid taps are grouped into one Google Sheets request after a short pause.
- The button stays responsive while cloud sync runs in the background.
